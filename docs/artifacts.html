Behavior: Capability Statements

The following artifacts define the specific capabilities that different types of systems are expected to have in order to comply with this implementation guide. Systems complying with the implementation guide are expected to declare conformance to one or more of the following capability statements.

Server conformance for Diagnostic Message Handling Server conformance for Diagnostic Message Handling
Server conformance for Encounter Message Handling Server conformance for Encounter Message Handling

Behavior: Message Definitions

These define the types of messages that can be sent and/or received by systems complying with this implementation guide

Notitification Admission Admission Notification - care provision started. FHIR equivalent of HL7v2 ADT^A01
Notitification Discharge Discharge Notification - patient handed over. FHIR equivalent of HL7v2 ADT^A03
Diagnostic Messaging Unsolicited transmission of an observation message. FHIR equivalent of HL7v2 ORU^R01
Patient Create Patient Create. FHIR equivalent of HL7v2 ADT^A28

These define new code systems used by systems conforming with this implementation guide
